# 客服系统设计方案

## 一、项目概述

### 1.1 项目背景
本项目旨在构建一个独立的客服系统，支持用户发起咨询、客服后台管理、消息实时收发、排队机制和多客服同步等功能。系统采用前后端分离架构，后端使用 Java 21 + Spring Boot 3.x，前端使用 React + Ant Design。最终将集成到现有 Java + Spring Boot 系统中。

### 1.2 项目模块
| 模块 | 说明 |
|------|------|
| customer-service | 后端服务模块 |
| customer-webapp | 前端应用模块 |

### 1.3 技术栈
- 后端：Java 21, Spring Boot 3.x, MyBatis-Plus, MySQL, Redis, WebSocket
- 前端：React 18, Ant Design 5.x, TypeScript
- 通信：HTTP REST API + WebSocket

### 1.4 后端技术规范

本项目后端开发严格遵循 Spring Boot 开发规范，使用 MyBatis-Plus 进行数据库交互。

#### 2.2.1 Controller层规范

```java
@RequestMapping("/api/conversation")
```

#### 2.2.2 Service层规范

```java
// 接口
public interface ConversationService extends IService<Conversation> {
    List<ConversationDTO> getQueueList();
    boolean takeConversation(Long id, Long agentId);
}

// 实现类
@Service
public class ConversationServiceImpl extends ServiceImpl<ConversationMapper, Conversation>
        implements ConversationService {
    @Override
    public List<ConversationDTO> getQueueList() {
        return this.list(new QueryLambdaWrapper<Conversation>()
            .eq(Conversation::getStatus, ConversationStatus.WAITING)
            .orderByAsc(Conversation::getQueuePosition))
            .stream()
            .map(this::convertToDTO)
            .collect(Collectors.toList());
    }
}
```

## 三、数据库设计

```sql
CREATE TABLE agent (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键',
    username VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
    password VARCHAR(255) NOT NULL COMMENT '密码',
    nickname VARCHAR(50) COMMENT '昵称',
    role TINYINT DEFAULT 1 COMMENT '角色: 1:普通客服 2:管理员',
    status TINYINT DEFAULT 1 COMMENT '在线状态: 1:在线 2:忙碌 3:离线',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'

```

#### 3.1.2 会话表 (conversation)
```sql
CREATE TABLE conversation (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键',
    visitor_id VARCHAR(64) NOT NULL COMMENT '访客ID',
    visitor_nickname VARCHAR(100) COMMENT '访客昵称',
    visitor_ip VARCHAR(64) COMMENT '访客IP',
    agent_id BIGINT COMMENT '接手客服ID',
    agent_nickname VARCHAR(50) COMMENT '客服昵称',
    status TINYINT DEFAULT 1 COMMENT '会话状态: 1:等待中 2:进行中 3:已结束 4:已评价',
    queue_position INT DEFAULT 0 COMMENT '排队位置',
    started_at TIMESTAMP NULL COMMENT '开始时间',
    ended_at TIMESTAMP NULL COMMENT '结束时间',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
);
```

#### 3.1.3 消息表 (message)
```sql
CREATE TABLE message (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键',
    conversation_id BIGINT NOT NULL COMMENT '会话ID',
```

## 六、业务流程

### 6.1 用户咨询流程

```
1. 用户打开网页 → 加载咨询浮窗组件
2. 用户点击浮窗 → 展开聊天窗口
3. 用户发送消息 → 创建会话（状态：等待中）
4. 会话加入排队队列 → 推送排队位置给用户
5. 客服接手 → 更新会话状态（状态：进行中）
6. 推送客服接入通知给用户 → 开始实时对话
7. 会话结束 → 用户可提交评价
```

### 6.2 客服工作流程

```
1. 客服登录 → 设置在线状态
2. 查看排队列表 → 选择等待中的用户
3. 点击接手 → 会话分配给自己
4. 与用户实时对话 → 发送/接收消息
5. 会话结束 → 可转接或结束会话
6. 继续处理下一个用户
```

### 6.3 排队机制

#### 6.3.1 排队规则
- 按用户加入时间先后顺序排列
- 有空闲客服（在线+非忙碌）时自动分配
- 客服可手动从列表中选择用户接手

#### 6.3.2 排队状态变化
- 用户加入 → 推送到所有在线客服
- 位置变化 → 实时更新给用户
- 被接手 → 从排队队列移除

### 6.4 多客服同步

#### 6.4.1 需要同步的内容
1. 客服在线状态变化
2. 排队队列变化
3. 会话接手/结束
4. 新消息通知
5. 转接请求

#### 6.4.2 同步方案
- Redis发布/订阅实现跨实例同步
- 数据库作为最终一致性保障

## 九、开发规范与验证

### 9.1 前端开发规范

前端开发需遵循 frontend-design skill 的指导原则：
- **设计美学**: 避免通用AI生成美学，选择独特、有辨识度的视觉方向
- **字体选择**: 使用有特色的字体，避免 Arial/Inter/Roboto 等通用字体
- **色彩与主题**: 使用 CSS 变量保持一致性，主色调搭配尖锐的 accent 色
- **动效**: 使用 CSS 动画或 Motion 库，注重高impact时刻
- **空间布局**: 尝试不对称、重叠、对角线流动等创意布局

### 9.2 后端开发规范

后端开发需遵循 backend-springboot-dev skill 的指导原则：
- **分层架构**: Controller → Service → Mapper → Entity
- **数据库交互**: 使用 MyBatis-Plus，优先使用 QueryLambdaWrapper
- **代码规范**: 单类不超过500行，单方法不超过80行
- **命名规范**: 类名大驼峰，变量名小驼峰，常量全大写

### 9.3 代码验证

每个子任务完成后必须执行编译验证：

| 模块 | 验证命令 | 说明 |
|------|----------|------|
| 前端 customer-webapp | `npm run build` | 验证 React/TypeScript 代码编译 |
| 后端 customer-service | `mvn clean compile` | 验证 Java 代码编译 |

验证通过后方可提交或进入下一任务。
