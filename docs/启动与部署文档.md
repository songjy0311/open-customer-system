## 项目概述

本项目为开源客服系统，包含两个模块：

| 模块 | 路径 | 说明 |
|---|---|---|
| `customer-service` | `/customer-service` | Spring Boot 后端服务，提供 REST API 和 WebSocket 通信 |
| `customer-webapp` | `/customer-webapp` | React + TypeScript 前端应用，包含客服工作台和访客聊天页面 |

## 本地开发启动

### 1. 基础设施准备

**启动 MySQL**（确保端口 `3308` 可用，或修改配置）：

```bash
# Docker 快速启动 MySQL（推荐）
docker run -d \
  --name customer-mysql \
  -e MYSQL_ROOT_PASSWORD=root123 \
  -p 3308:3306 \
  mysql:8.0

# 若使用本地 MySQL，默认端口为 3306，需修改 application.yml 中的端口
```

**启动 Redis**（确保端口 `6371` 可用，或修改配置）：

```bash
# Docker 快速启动 Redis（推荐）
docker run -d \
  --name customer-redis \
  -p 6371:6379 \
  redis:6.0

# 若使用本地 Redis 默认端口 6379，需修改 application.yml 中的端口
```

### 2. 数据库初始化

连接 MySQL 后执行初始化脚本：

```bash
mysql -h 127.0.0.1 -P 3308 -u root -proot123 < customer-service/src/main/resources/schema.sql
```

或手动登录 MySQL 执行：

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS customer_service_mobile
  DEFAULT CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

-- 切换数据库后执行 schema.sql 中的建表语句
USE customer_service_mobile;
-- ... 执行其余 DDL 及初始化数据
```

> ⚠️ **注意**：`schema.sql` 第 5 行为 `USE customer_service;`，存在笔误，实际数据库名为 `customer_service_mobile`，执行时请使用 `USE customer_service_mobile;`。

### 3. 启动后端服务

```bash
cd customer-service

# 使用 Maven Wrapper（若已配置）
./mvnw spring-boot:run

# 或指定 Maven 路径
mvn spring-boot:run

# 或先编译再运行
mvn compile
mvn spring-boot:run
```

启动成功后，后端服务监听在：`http://localhost:8080`

**验证启动**：

```bash
curl http://localhost:8080/api/auth/ping
# 或访问任意接口确认服务正常
```

### 4. 启动前端服务

```bash
cd customer-webapp

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

启动成功后，前端服务监听在：`http://localhost:3000`

## 配置说明

### 后端配置

配置文件路径：`customer-service/src/main/resources/application.yml`

```yaml
server:
  port: 8080                    # 服务端口，可按需修改

spring:
  datasource:
    url: jdbc:mysql://localhost:3308/customer_service_mobile  # MySQL 连接地址
    username: root              # MySQL 用户名
    password: root123           # MySQL 密码

  data:
    redis:
      host: localhost           # Redis 地址
      port: 6371                # Redis 端口（注意非默认 6379）
      database: 0               # Redis 数据库编号

jwt:
  secret: kefu-customer-system-secret-key-2024  # JWT 签名密钥，生产环境务必修改
  expiration: 86400000          # JWT 过期时间（毫秒），默认 24 小时
```

**生产环境必须修改的配置项：**

| 配置项 | 说明 |
|---|---|
| `spring.datasource.password` | 数据库密码，使用强密码 |
| `jwt.secret` | JWT 签名密钥，使用随机长字符串 |
| `spring.data.redis.password` | 若 Redis 有密码需添加此项 |

### 前端配置

配置文件路径：`customer-webapp/vite.config.ts`

```typescript
server: {
  port: 3000,           // 前端开发服务器端口
  proxy: {
    '/api': {
      target: 'http://localhost:8080',   // 代理到后端 REST API
      changeOrigin: true
    },
    '/ws': {
      target: 'ws://localhost:8080',     // 代理到后端 WebSocket
      ws: true
    }
  }
}
```

若后端地址发生变更，修改 `target` 的值即可。

## 生产环境部署

### 后端打包部署

```bash
cd customer-service

# 打包为可执行 JAR（跳过测试）
mvn package -DskipTests

# 生成文件位于 target/customer-service-1.0.0.jar
# 启动命令
java -jar target/customer-service-1.0.0.jar

# 生产环境推荐指定配置（外部配置文件覆盖）
java -jar target/customer-service-1.0.0.jar \
  --spring.datasource.url=jdbc:mysql://your-db-host:3306/customer_service_mobile \
  --spring.datasource.password=your-strong-password \
  --jwt.secret=your-random-secret-key \
  --server.port=8080
```

**后台运行（nohup）：**

```bash
nohup java -jar target/customer-service-1.0.0.jar > logs/app.log 2>&1 &
echo $! > app.pid
```

**停止服务：**

```bash
kill $(cat app.pid)
```

### 前端打包部署

```bash
cd customer-webapp

# 安装依赖
npm install

# 构建生产包
npm run build

# 构建产物位于 dist/ 目录，将其部署到 Web 服务器（Nginx 等）
```

### Nginx 配置参考

将前端静态文件与后端 API 通过 Nginx 统一代理：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    root /path/to/customer-webapp/dist;
    index index.html;

    # 前端路由（React Router）支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 代理后端 REST API
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # 代理 WebSocket
    location /ws/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 86400;
    }
}
```

## 常见问题

### Q1：启动后端报 `Communications link failure`

**原因**：无法连接 MySQL。

**排查步骤**：
1. 确认 MySQL 已启动：`docker ps | grep mysql`
2. 确认端口正确：配置文件默认端口为 `3308`，非标准 MySQL 端口 `3306`
3. 确认账号密码正确：`mysql -h 127.0.0.1 -P 3308 -u root -proot123`

---

### Q2：启动后端报 `Connection refused` (Redis)

**原因**：无法连接 Redis。

**排查步骤**：
1. 确认 Redis 已启动：`docker ps | grep redis`
2. 确认端口正确：配置文件默认端口为 `6371`，非标准 Redis 端口 `6379`
3. 测试连通性：`redis-cli -h 127.0.0.1 -p 6371 ping`

---

### Q3：前端启动后请求 404

**原因**：Vite 代理已配置，通常是后端未启动。

**排查步骤**：
1. 确认后端已在 `8080` 端口正常运行
2. 检查 `vite.config.ts` 中 `proxy.target` 地址与后端一致

---

### Q4：WebSocket 无法连接

**原因**：WebSocket 握手失败或跨域问题。

**排查步骤**：
1. 开发环境下 WebSocket 通过 Vite 代理，确保后端正常运行
2. 生产环境下确认 Nginx 配置了 `Upgrade` 和 `Connection` header
3. 后端 `WebSocketConfig` 已配置 `setAllowedOrigins("*")`，允许跨域

---

### Q5：`schema.sql` 执行报错 `Unknown database 'customer_service'`

**原因**：`schema.sql` 第 5 行存在笔误 `USE customer_service;`，实际库名为 `customer_service_mobile`。

**解决**：手动执行时将该行改为 `USE customer_service_mobile;`，或先手动创建数据库再执行建表语句。
